# NPM 依赖解析和预构建

原生 ES 是不支持裸模块导入的

import { mapGetters } from "Vuex";
1

浏览器在执行上面 👆 代码时会抛出一个错误。Vite会检查所有被加载的源文件中的裸模快导入并进行解析。

  1. 预结构 可提高页面加载速度，并将 CommonJS / UMD 转换为 ESM 格式。
  2. 重写导入为合法的 URL。例如 /node_modules/.vite/my-dep.js?v=f3sf2ebd

# 热模块重载

Vite 提供了一套原生的 ESM 的HMR API,该 API 提供及时、准确的更新，无需刷新页面或清除应用程序状态。 通过 Vite 创建的项目无需手动配置该项。

# CSS

导入.css文件将会把内容插入到<style>标签中，同时也带有 HMR 的支持。

# @import 内联与变基

Vite 通过postcss-import与配置支持了 CSS 的@import内联。路径别名也遵从 CSS@import。换句话说，所有 CSS url() 引用，即使导入的文件在不同的目录中，也总是自动变基，以确保正确性。 支持 Sass 和 Less 文件

# CSS Modules

任何.modules.css后缀的文件都被认为是一个 CSS Modules 文件，导入这样的文件会返回一个模块对象。

/* example.module.css */
.skybule {
  color: "skyblue";
}

# CSS 与处理器

Vite 没有对 CSS 预处理器进行与配置，但提供了.scss, .sass, .less, .styl 和 .stylus 文件的内置支持。需手动安装对应的依赖。

# .scss and .sass
npm install -D sass

# .less
npm install -D less

# .styl and .stylus
npm install -D stylus

# 构建优化

TIP

以下罗列的会自动应用为构建过程的一部分，无需配置，除非禁用。

# CSS 代码分割

Vite 会自动将一个异步 chunk 模块中 CSS 代码单独抽取为一个文件，该文件会在 chunk 模块加载完成时自动通过link标签载入。 可通过配置build.cssCodeSplit为 false 来禁用分割。

# 预加载指令

Vite 会在打包输出的 HTML 文件中引入<link rel="modulepreload">进行预加载出来。
