# 同步代码与异步代码

定时器属于异步代码。

  1. 同步代码最先执行。
  2. 同步代码执行完成后执行代码才开始计时执行。
  3. 异步代码根据时间或触发先后循序执行。

# setTimeout

延迟执行。本质是去异步队列注册一个任务，并反该唯一任务id，随着注册次数的增加而增加。 可通过 clearTimeout 方法传入任务 id 进行定时器的清除。

/**
 * 参数说明：
 *  1：回调函数
 *  2：延迟执行时间，毫秒值。
 */
const timer = setTimeout (() => {}, 3000)

// 清除定时器
clearTimeout(timer)

# setInterval

每间隔一段时间执行一次。同 setTimeout。间隔时间必须大于等于 1，不能是 0。 可通过 clearInterval 方法传入任务 id 进行定时器的清除。 做动画欠佳，可能会出现卡顿、掉帧、延迟。

/**
 * 参数说明：
 *  1：回调函数
 *  2：间隔时间，毫秒值。
 */
const timer = setInterval (() => {}, 3000)

// 清除定时器
clearInterval(timer)

# 回调函数的this

默认指向 window。

改变回调函数的 this：

  1. bind 强制改变 this。
const obj = {
  na me: 'this.name'
}
function fun (cb) {
  cb()
}
fun ((function () {
  console.log(this.name) // this.name
}).bind(obj)) // bind 返回一个函数

# 请求动画帧

requestAnimationFrame 等待一次渲染，在下次渲染时继续执行。严格的与页面刷新频率一致。

function fn () {
  console.log('半导体明天大涨~')
  requestAnimationFrame (fn)
}
fn()
